# Typst 最佳实践

> 何谓最佳？看上去比较舒服🐸如果你觉得不舒服，随意调整即可

自认为比较合理的 Typst 项目目录结构，能够让各个部分清晰明了，让内容和样式分离：优化用户使用体验，让模板开发更加清晰（通过目录结构来强制分离关注点）

```txt
/项目根目录
│
├── main.typ              <-- 入口文件（几乎没什么内容，变得很短、很干净）
├── template.typ          <-- 模板逻辑（这里定义具体的文档结构并引入样式）
├── ref.bib               <-- 参考文献库
│
├── contents/             <-- 专门放各种长内容的文件夹
│   ├── abstract.typ      <-- 存放摘要
│   ├── task.typ          <-- 存放人员分工
│   └── appendix.typ      <-- 存放附录、问卷、图片
│
└── style/                <-- 样式文件夹
    ├── heading.typ
    └── ...
```
